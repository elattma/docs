title: "Overriding a Query"
description: A definition for the Overrides object
---

## Properties

<ResponseField name="concepts" type="string[]">
  A list of semantically important concepts the desired context is related to.
</ResponseField>
<ResponseField name="entities" type="string[]">
  A list of case-insensitive entity names to filter on. An entity is a person, place, or organization that is
  mentioned in the text. 
</ResponseField>
<ResponseField name="time_start" type="int">
  A unix timestamp in seconds that represents the start of the time range to filter on.
</ResponseField>
<ResponseField name="time_end" type="int">
  A unix timestamp in seconds that represents the end of the time range to filter on.
</ResponseField>
<ResponseField name="time_sort" type="string">
  A string that represents the order to sort the results in. Can be either `asc` or `desc`.
</ResponseField>
<ResponseField name="limit" type="int">
  The maximum number of results to return.
</ResponseField>
<ResponseField name="integration_types" type="string[]">
  A list of integration types to filter on. Supported types include: 'crm', 'customer_support', 'document', 'email', 'code', 'upload', 'website'.
</ResponseField>
<ResponseField name="search_method" type="string">
  The method to use when searching for results. Supported methods include: 'exact', 'relevant'.
</ResponseField>
<ResponseField name="bypass_agent" type="boolean">
  Whether or not to bypass the agent when searching for results.
</ResponseField>
